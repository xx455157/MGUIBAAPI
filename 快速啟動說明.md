# 🚀 MGUIBAAPI 快速啟動指南

## ✨ 類似 VS2022 的 Debug 功能已配置完成！

現在你可以在 Cursor 中像使用 Visual Studio 2022 一樣啟動和調試 API。

## 🎯 三種啟動方式

### 方式 1: 按 F5 快捷鍵（最簡單）⭐

1. 在 Cursor 中打開 `MGUIBAAPI` 專案
2. **直接按 `F5` 鍵**
3. API 會自動：
   - ✅ 編譯專案
   - ✅ 啟動服務
   - ✅ 自動打開 Swagger 文檔

**就是這麼簡單！**

### 方式 2: 使用調試面板

1. 點擊左側的 **🐛 調試圖標** (或按 `Ctrl+Shift+D`)
2. 在頂部選擇配置：
   - **`.NET Core Launch (API)`** - 標準模式
   - **`.NET Core Launch (API - Watch Mode)`** ⭐ - 自動重啟模式（推薦）
3. 點擊 **▶️ 綠色播放按鈕** 或按 `F5`

### 方式 3: 使用命令面板

1. 按 `Ctrl+Shift+P`
2. 輸入 `Debug: Start Debugging`
3. 選擇配置並啟動

## 🔥 Watch Mode（自動重啟模式）- 強烈推薦！

選擇 **`.NET Core Launch (API - Watch Mode)`** 配置後：

- ✅ 修改任何 `.cs` 檔案並保存
- ✅ 自動重新編譯
- ✅ 自動重啟 API
- ✅ 無需手動停止和啟動
- ✅ 支援斷點調試

**就像 VS2022 的熱重載功能！**

## 🐛 斷點調試

### 設置斷點
- 在程式碼行號左側點擊，出現 **紅色圓點** ⚫
- 或按 `F9` 切換斷點

### 調試快捷鍵
- **F5**: 繼續執行 / 啟動調試
- **F9**: 切換斷點
- **F10**: 單步執行（不進入函數）
- **F11**: 單步進入（進入函數內部）
- **Shift+F11**: 單步跳出
- **Shift+F5**: 停止調試
- **Ctrl+Shift+F5**: 重新啟動

### 調試功能
- **變數面板**: 查看當前作用域的所有變數
- **監視**: 添加表達式監視
- **調試控制台**: 執行 C# 表達式
- **呼叫堆疊**: 查看函數呼叫鏈

## 📋 配置說明

### 已創建的檔案

1. **`.vscode/launch.json`** - 調試配置
   - 標準啟動模式
   - Watch Mode（自動重啟）
   - 附加到進程

2. **`.vscode/tasks.json`** - 構建任務
   - Build 任務
   - Watch 任務
   - Publish 任務
   - Clean 任務

### 服務資訊

- **端口**: http://localhost:23232
- **Swagger**: http://localhost:23232/swagger
- **環境**: Development

## ⚠️ 前置需求

### 1. 安裝 C# 擴展

如果還沒有安裝，請：

1. 在 Cursor 中按 `Ctrl+Shift+X` 打開擴展面板
2. 搜尋 `C#`
3. 安裝 **C#** (由 Microsoft 提供，ID: `ms-dotnettools.csharp`)

或使用命令：
```
ext install ms-dotnettools.csharp
```

### 2. 確認 .NET Core SDK

確認已安裝 .NET Core 2.2 SDK（或更高版本）

## 💡 使用建議

### 開發時
1. 選擇 **Watch Mode** 配置
2. 按 `F5` 啟動
3. 修改程式碼並保存
4. 自動重啟，立即看到效果

### 調試時
1. 設置斷點
2. 選擇標準配置
3. 按 `F5` 啟動
4. 使用調試工具列進行調試

### 測試時
1. 選擇標準配置
2. 按 `F5` 啟動
3. 在 Swagger 中測試 API

## 🐛 常見問題

### Q: 按 F5 沒有反應？
A: 確認已安裝 C# 擴展，並在專案根目錄打開 Cursor

### Q: 斷點無法命中？
A: 確認程式碼已編譯為 Debug 模式，斷點設置在可執行的程式碼行上

### Q: 端口被佔用？
A: 停止其他佔用 23232 端口的服務，或修改 `launch.json` 中的端口

### Q: Watch Mode 沒有自動重啟？
A: 確認選擇的是 Watch Mode 配置，且檔案已保存
